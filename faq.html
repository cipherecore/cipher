Frequently Asked Questions

Find answers to common questions about CiphereCore features, security, and usage.

Getting Started

Is CiphereCore really free?
▼
Yes, CiphereCore is 100% free. No subscription fees, no"freemium" catches, and no account required to download or use it. We believe security should be accessible to everyone.
What operating systems are supported?
▼
CiphereCore currently supports Windows (7 and later). We're actively working on Linux and macOS versions. Stay tuned for announcements.
Do I need Python to run CiphereCore?
▼
No. CiphereCore is distributed as a standalone Windows installer (.exe) with all dependencies bundled. Simply download and run—Python is not required.
How much storage does CiphereCore require?
▼
The installer is approximately 32MB (cipherecore_installer.exe), portable is 30MB (cipherecore.exe), and ZIP archive is 82MB. After installation, CiphereCore uses minimal disk space (~50MB including user data, logs, and local database).

Encryption & Security

Which encryption algorithms does CiphereCore use?
▼
CiphereCore supports military-grade algorithms: AES-256 (recommended), Fernet, ChaCha20, Triple DES, and AES-CTR. AES-256 uses industry standard CBC mode with PKCS7 padding and random initialization vectors.
Is my data safe? How secure is the encryption?
▼
Yes. CiphereCore uses battle-tested, open-source cryptography libraries. The encryption algorithms are NIST-approved and have undergone extensive peer review. We don't use proprietary encryption—what you see is what you get.
What is the Shield Scan feature?
▼
Shield Scan is an automated malware detection system for steganographic operations. It scans images for embedded PHP/JS code, header corruption, and injection attacks before processing. This protects you from hidden threats in supposedly innocent image files.
Can I decrypt files if I forget my password?
▼
No. This is intentional for security. CiphereCore has no "master key" or recovery method—your password IS the key. If you forget it, the encrypted data remains inaccessible. Store your passwords securely using the Password Generator.

Chat & Messaging

How does the encrypted chat work?
▼
CiphereCore offers three chat modes: Server Chat (connect to a community server), Global Chat (broadcast messages via MongoDB), and Private Messages (direct 1-on-1 encrypted messaging). All messages between users are end-to-end encrypted.
Do I need MongoDB to use CiphereCore?
▼
MongoDB is optional. CiphereCore works fully offline using local SQLite storage. However, for Global Chat, Private Messages, and Friend System features, MongoDB enables cloud persistence and syncing across devices.
Can I host my own CiphereCore server?
▼
Yes! CiphereCore includes a built-in server mode. Navigate to Chat → Server, set a port and optional password, then "Start Server". Share your IP and port with others to connect. Perfect for private team communication.

Privacy & Data

Does CiphereCore collect data about me?
▼
No data collection whatsoever. CiphereCore runs locally on your machine. We don't have analytics, telemetry, or tracking. Your encrypted files, messages, and user data never leave your device unless you explicitly configure MongoDB sync.
How are passwords stored?
▼
User passwords are hashed using bcrypt with 12 rounds of salt—a modern, computationally expensive algorithm resistant to brute-force attacks. Even if someone obtained the database, they couldn't recover the original passwords.
Is the source code available?
▼
CiphereCore's cryptographic core uses open-source libraries verified by the security community. The application logic is proprietary for now, but we're committed to progressive transparency as the project grows.

Files & Steganography

What file types can I encrypt?
▼
All file types are supported: documents, images, videos, archives, executables—anything. You can encrypt files up to 500MB. Encrypted files are saved with a .enc extension.
How does image steganography work?
▼
Steganography hides text messages inside image files (PNG, JPG, BMP) without visibly altering the image. The recipient needs CiphereCore and the extraction password to reveal the hidden message. Perfect for covert communication.
Can steganography be detected?
▼
High-quality steganography with proper techniques is nearly impossible to detect without extraction. CiphereCore uses advanced algorithms to embed data while maintaining image integrity, making detection extremely difficult.
